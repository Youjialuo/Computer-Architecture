# 时序电路 — RS锁存器、D锁存器、D触发器、多位寄存器 完全详解

---

## 〇、时序电路 vs 组合电路（先分清门派）

| 维度 | 组合电路 (Combinational) | 时序电路 (Sequential) |
|------|-------------------------|------------------------|
| 输出依赖 | **仅当前输入** | **当前输入 + 历史状态** |
| 有无记忆 | ❌ 无 | ✅ 有（状态保存在触发器中） |
| 有无时钟 | 无 | 有（同步时序电路） |
| 典型器件 | 与/或/非门、加法器、译码器 | 锁存器、触发器、寄存器、计数器 |
| 函数形式 | Y = f(X) | Y(t) = f(X(t), Q(t)), Q(t+1) = g(X(t), Q(t)) |

> **核心思想**：时序电路引入**反馈**回路，把输出接回输入端，从而"记住"之前的信号。锁存器是所有时序电路的基础"原子"。

## 一、RS 锁存器（RS Latch）— 基本存储单元

### 1.1 电路结构（NOR门实现）

```
         ┌────────────┐
   R ────┤NOR1        ├──── Q
         │        ┌───┘
         │   ┌────┘
         └───┼────┐
             │    │
         ┌───┘    │
         │   ┌────┘
   S ────┤NOR2    │
         │        └──── Q̅ (Q_bar)
         └────────────┘
```

**连接方式（口诀）：**
- S → NOR2 输入，R → NOR1 输入
- NOR1 输出 = Q，NOR2 输出 = Q̅
- **交叉反馈**：Q 接回 NOR2 的另一个输入，Q̅ 接回 NOR1 的另一个输入

> ⚠️ 这是因为 NOR 门特性：**有1出0，全0出1**。

### 1.2 四种工作状态深度分析

#### 状态一：S=0, R=0 — 保持（Hold）

```
假设当前 Q=1, Q̅=0：
  NOR1: 输入 R=0, Q̅=0 → 全0 → 输出Q=1 ✅ 保持
  NOR2: 输入 S=0, Q=1 → 有1 → 输出Q̅=0 ✅ 保持

假设当前 Q=0, Q̅=1：
  NOR1: 输入 R=0, Q̅=1 → 有1 → 输出Q=0 ✅ 保持
  NOR2: 输入 S=0, Q=0 → 全0 → 输出Q̅=1 ✅ 保持
```

> **结论**：S=0, R=0 时，无论初始Q是什么，都维持不变。这就是"记忆"的本质。

---

#### 状态二：S=1, R=0 — 置位（Set）

```
无论初始Q是什么：
  NOR2: 输入 S=1, Q=x → 有1必出0 → Q̅=0 (立即！)
  NOR1: 输入 R=0, Q̅=0 → 全0 → Q=1
  
最终：Q=1, Q̅=0，完成"Set"
```

> **S = Set = 置1**。就像把开关打到"开"。

---

#### 状态三：S=0, R=1 — 复位（Reset）

```
无论初始Q是什么：
  NOR1: 输入 R=1, Q̅=x → 有1必出0 → Q=0
  NOR2: 输入 S=0, Q=0 → 全0 → Q̅=1
  
最终：Q=0, Q̅=1，完成"Reset"
```

> **R = Reset = 清0**。就像把开关打到"关"。

---

#### 状态四：S=1, R=1 — 禁止（Forbidden）❌

```
  NOR1: 输入 R=1, Q̅=x → Q=0
  NOR2: 输入 S=1, Q=x → Q̅=0
  → Q=0, Q̅=0（两个输出相同！违反了互补原则）
  
更致命的是：当S和R同时从1回到0时：
  两个NOR的输入都变为(0,0)→(0,0)，输出会振荡/不确定！
  竞争条件 (Race Condition) → 谁先变？不知道！
```

> ⚠️ **RS锁存器第一铁律：S和R绝对不能同时为1！**

---

### 1.3 真值表（特性表）

| S | R | Q(t+1) | Q̅(t+1) | 操作 | 说明 |
|---|----|--------|---------|------|------|
| 0 | 0 | Q(t) | Q̅(t) | **保持** | 记忆状态 |
| 0 | 1 | 0 | 1 | **Reset** | 清0 |
| 1 | 0 | 1 | 0 | **Set** | 置1 |
| 1 | 1 | 0 | 0 | **禁止** | 不稳定！ |

### 1.4 NAND门实现的RS锁存器（̅S̅R̅ 锁存器）

用NAND门也可以搭建RS锁存器，但输入变为**低有效**（记为 ̅S̅ 和 ̅R̅）：

```
         ┌────────────┐
   ̅S̅ ───┤NAND1       ├──── Q
         │        ┌───┘
         │   ┌────┘
         └───┼────┐
             │    │
         ┌───┘    │
         │   ┌────┘
   ̅R̅ ───┤NAND2
         │        └──── Q̅
         └────────────┘
```

| ̅S̅ | ̅R̅ | Q(t+1) | 操作 |
|------|------|--------|------|
| 1 | 1 | Q(t) | 保持 |
| 1 | 0 | 0 | Reset |
| 0 | 1 | 1 | Set |
| 0 | 0 | 1,1 | **禁止** |

> ⚠️ NAND版本铁律：**̅S̅ 和 ̅R̅ 不能同时为 0！**（刚好和NOR版本反过来）

### 1.5 RS锁存器的时序波形分析（★重难点）

```
R: ____----________----____
S: ________----____--------
   
Q: ____----__________----__
Q̅: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
         ↑    ↑     ↑    ↑
        Set  保持  Reset  Set
```

**时序分析要点：**
- S从0→1的时刻：Q经过一个门延迟后变为1（Set）
- S回到0后：Q保持1不变
- R从0→1的时刻：Q经过一个门延迟后变为0（Reset）
- 传播延迟 ≈ **2个门延迟**（经过两次NOR）

## 二、时钟控制的RS锁存器（RS Clocked Latch / "RS Time Lock"）

用户提到的"RS time lock"指的是带**时钟使能**的RS锁存器。

### 2.1 电路结构

```
           ┌───┐
  CLK ──┬──┤AND├──→ NOR RS锁存器的S端
  S ────┘  └───┘
         
           ┌───┐
  CLK ──┬──┤AND├──→ NOR RS锁存器的R端
  R ────┘  └───┘
```

**工作原理：**
- CLK=0 → 两个AND输出恒为0 → RS锁存器保持（相当于S=0,R=0）
- CLK=1 → S和R直接传递到RS锁存器 → 正常Set/Reset操作

> 时钟 = 门控信号（Gating Signal），只在时钟有效时才允许改变状态。

### 2.2 时序图

```
CLK: ‾‾‾‾‾‾‾‾‾‾‾‾____‾‾‾‾‾‾‾‾‾‾‾‾
S:   ------________----------------
R:   ____________------____________

Q:   ------______________________--  
                 ↑锁存器的输出只在CLK=1时能改变
```

## 三、D 锁存器（D Latch）— 透明锁存器

### 3.1 从RS锁存器到D锁存器（消除禁止状态）

**问题**：RS锁存器有S=R=1的禁止状态。
**解法**：让S和R始终互补 → 用单个输入端 D + 反相器驱动 R = ¬D。

```
              ┌────────────┐
  D ──┬───────┤AND (门控)  ├──→ NOR RS锁存器的S
      │       └────────────┘
      │   ┌─┐
      └───┤1│ (反相器)   
          └┬┘       ┌────────────┐
           └────────┤AND (门控)  ├──→ NOR RS锁存器的R
                    └────────────┘
              ↑
             CLK ────────────────┘ (共享)
```

**化简后的等效电路：**

```
  D ────────┬────────────┐
            │   ┌───┐    │
  CLK ──────┴───┤AND├────┤── Q
                └───┘    │
                    ┌────┘
                    │
              ┌───┐ │  ┌───┐
              │NOT├─┴──┤AND├── Q̅
              └───┘    └───┘
```

### 3.2 工作特性

| CLK | 行为 | 说明 |
|-----|------|------|
| 0 | Q保持 | 门关闭，不响应输入 |
| 1 | Q = D | **透明模式**：输出跟随输入 |

> ⚠️ **"透明"的含义**：CLK=1期间，D怎么变，Q就怎么变。就像一扇透明的窗户——你能直接看到输入。

### 3.3 D锁存器的致命弱点

```
CLK: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾____
D:   ____--__------____--________

Q:   ____--__------____--________
         ↑ ↑      ↑    ↑ ↑
       每个毛刺都透传！毛刺期间采样不稳定！
```

> ⚠️ **电平敏感导致的问题**：CLK=1期间的任何D的变化都会立刻传到Q。如果D上有毛刺（glitch），Q就会跟着抖，无法稳定采样。

## 四、D 触发器（D Flip-Flop）— 边沿触发（★核心器件）

### 4.1 为什么需要D触发器？

D锁存器的透明特性导致它在复杂数字系统中产生问题：
- 多个锁存器级联时，如果都用同一个时钟，信号会在一个CLK周期内"穿越"多级
- 无法实现稳定的同步采样

**D触发器的解决方案**：只在时钟**边沿**（上升沿↑或下降沿↓）采样一次，其余时间锁死。

### 4.2 主从结构（Master-Slave D Flip-Flop）

```
        主锁存器             从锁存器
     ┌───────────┐      ┌───────────┐
  D──┤ D Latch   ├──────┤ D Latch   ├── Q
     │ (Master)  │      │ (Slave)   │
     └──┬────────┘      └──┬────────┘
        │  CLK             │  ¬CLK
        │                  │
   CLK──┴──────────────────┴── (反相后给Slave)
```

**工作过程（三阶段）：**

| 阶段 | CLK | Master | Slave | 输出Q | 
|------|-----|--------|-------|-------|
| ① | 0 | 透明（跟D） | 锁存 | **保持旧值** | 
| ② | ↑上升沿 | 锁存（采样D） | 还在锁存 | **保持旧值**（还没变） | 
| ③ | 1 | 锁存（不再跟D） | 透明（跟Master输出） | **更新为新值** |

> **核心机制**：时钟上升沿时，Master锁存D的值；然后在下半个CLK周期，Slave把Master锁存的值输出。D的信号在上升沿之后怎么变都不会影响Q！

### 4.3 D触发器的真值表（特性表）

| CLK | D | Q(t+1) | 说明 |
|-----|---|--------|------|
| ↑ (上升沿) | 0 | 0 | 采样到0 |
| ↑ (上升沿) | 1 | 1 | 采样到1 |
| 0 / 1 / ↓ | × | Q(t) | **保持不变** |

**特征方程：Q(t+1) = D**（仅发生在时钟有效边沿）

### 4.4 D触发器的时序图

```
CLK: __‾‾--‾‾__‾‾--‾‾__‾‾--‾‾__
D:   ----______--------__--------
     
Q:   ________----____________----
           ↑              ↑
      上升沿采样D=0   上升沿采样D=1
      
      D在时钟边沿之后的变化
      不影响Q！← 和锁存器的关键区别
```

### 4.5 带异步Set/Reset的D触发器（★常见题型）

很多实际D触发器带有异步（不受CLK控制）的Set和Reset引脚：

```
         ┌─────────────┐
   CLK ──┤▷            │
    D  ──┤D          Q ├── Q
   ̅P̅R̅E̅ ─┤PRE         ̅Q̅├── Q̅
   ̅C̅L̅R̅ ─┤CLR           │
         └─────────────┘
```

| ̅P̅R̅E̅ | ̅C̅L̅R̅ | CLK | 行为 |
|------|------|-----|------|
| 0 | 1 | × | Q=1（异步置位，不管CLK）|
| 1 | 0 | × | Q=0（异步清零，不管CLK）|
| 0 | 0 | × | **禁止**（Q=Q̅=1）|
| 1 | 1 | ↑ | 正常D触发 |

> ⚠️ PRE和CLR通常是**低有效**（带横杠），且不能同时有效。

## 五、四位D触发器——基础寄存器

### 5.1 4位寄存器（并行寄存器）

将4个D触发器**并联**，共享同一时钟和同一控制信号：

```
  D3 ──┬──┤D     Q├── Q3
       │  │  FF3  │
  D2 ──┼──┤D     Q├── Q2
       │  │  FF2  │
  D1 ──┼──┤D     Q├── Q1
       │  │  FF1  │
  D0 ──┴──┤D     Q├── Q0
          │  FF0  │
  CLK ────┤▷      │  (所有FF共享同一时钟)
          └───────┘
```

**工作原理**：时钟上升沿到达时，4个D触发器**同时**采样各自的D输入，把D[3:0]锁存到Q[3:0]。

**符号表示：**
```
      ┌────────┐
D[3:0]┤        ├─ Q[3:0]
      │  4位   │
CLK ──┤▷ 寄存器├─ 
      │        │
EN  ──┤ EN     │  (可选使能)
      └────────┘
```

### 5.2 带使能的寄存器

```
  EN ──┬───┐
       │   ├──→ 每个FF前的MUX:
  D ───┼───┤   当EN=0 → 选Q(自己的输出)→ 保持
       │   │   当EN=1 → 选D(外部输入) → 更新
  CLK ─┘   └──→
```

> 使能端（EN, Enable）：EN=0时，即使时钟沿到达也不更新。

## 六、移位寄存器（Shift Register）

### 6.1 4位移位寄存器（串入-串出 SISO / 串入-并出 SIPO）

不共享输入，而是**级联**：每个FF的Q接下一个FF的D：

```
 串行输入
  Sin ──→ D  Q0 ──→ D  Q1 ──→ D  Q2 ──→ D  Q3 ──→ 串行输出
             ▷          ▷          ▷          ▷
              ↑          ↑          ↑          ↑
             └──────────┴──────────┴──────────┘
                         CLK (共享)
```

**移位过程**（Sin=1011，初始全0）：

| CLK↑ | Q0 | Q1 | Q2 | Q3 | 说明 |
|------|----|----|----|----|------|
| 0 | 0 | 0 | 0 | 0 | 初始 |
| 1 | 1 | 0 | 0 | 0 | 移入1 |
| 2 | 0 | 1 | 0 | 0 | 移入0，1右移 |
| 3 | 1 | 0 | 1 | 0 | 移入1 |
| 4 | 1 | 1 | 0 | 1 | 移入1，数据全部进入 |

> 4个CLK周期后，4位数据完全进入寄存器。

### 6.2 并出 vs 串出

```
              并出 (并行输出所有位)
              ↓ ↓ ↓ ↓
Sin──→ D→Q ─→ D→Q ─→ D→Q ─→ D→Q ──→ Sout (串行输出一位一位出)
          ▷      ▷      ▷      ▷
```

### 6.3 寄存器符号速查

| 类型 | 符号 | 数据流 |
|------|------|--------|
| 并行寄存器 | 矩形块，D[n:0]入，Q[n:0]出，CLK | 并行入/并行出 |
| 移位寄存器 SISO | 矩形块，Sin入，Sout出，CLK | 串行入/串行出 |
| 移位寄存器 SIPO | 矩形块，Sin入，Q[n:0]出，CLK | 串行入/并行出 |
| 移位寄存器 PISO | 矩形块，D[n:0]入(Load)，Sout出，CLK | 并行入/串行出 |

## 七、综合对比表（★考前必看）

| 特性 | RS锁存器 | D锁存器 | D触发器 | 寄存器 |
|------|----------|---------|---------|--------|
| **触发方式** | 异步(直接) | 电平(CLK=1) | **边沿(↑或↓)** | 边沿 |
| **采样时机** | R/S一变化 | CLK=1期间任意时刻 | **仅时钟边沿** | 仅时钟边沿 |
| **毛刺敏感** | 无时钟 | ⚠️ 敏感 | ✅ 免疫 | ✅ 免疫 |
| **基本组成** | 2×NOR | 1×RS+1×反相器 | 2×D锁存器(主从) | n×D触发器 |
| **输入端** | R, S | D, CLK | D, CLK | D[n:0], CLK |
| **特征方程** | Q+ = S + Q·¬R | Q+ = D (CLK=1时) | Q+ = D (上升沿) | Q+[n:0] = D[n:0] |
| **能级联吗** | 不能(不共享时钟) | ⚠️ 谨慎(穿越问题) | ✅ 可以 | ✅ 可以 |

## 九、重难点总结（★必背）

### 必须区分的概念对

| 概念A | 概念B | 关键区别 |
|-------|-------|----------|
| 锁存器 (Latch) | 触发器 (Flip-Flop) | **电平触发 vs 边沿触发** |
| NOR RS锁存器 | NAND RS锁存器 | 高有效 vs 低有效，禁止状态不同 |
| 主锁存器 (Master) | 从锁存器 (Slave) | 输入采样 vs 输出保持 |
| 同步置位/复位 | 异步置位/复位 | 受CLK控制 vs 不受CLK控制 |
| 并行寄存器 | 移位寄存器 | 同时采样 vs 逐位移动 |
| 透明传输 | 边沿锁存 | CLK=1期间跟随 vs 仅边沿瞬间 |

### 核心口诀

> **RS锁存器**："S置1 R清0，11禁止记心中"
>
> **D锁存器**："时钟一开就透明，毛刺抖动不稳定"
>
> **D触发器**："主从两级分两边，上升沿才采一点"
>
> **寄存器**："并排一堆D触发，共享时钟同时抓"
>
> **移位寄存器**："级联首尾接一串，每个时钟移一位"
