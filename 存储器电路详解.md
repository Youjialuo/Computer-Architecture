# 存储器电路详解 —— 计算机组成原理专家级整理

## 一、存储器的基本概念与冯·诺依曼体系定位

### 1.1 存储程序（Stored Program）概念

> 来自课件：*"程序和数据以二进制形式存储在同一存储器中，计算机通过改变存储器内容来改变程序。"*

这是冯·诺依曼架构的**核心思想**：
- 指令和数据放在**同一个**存储器中
- CPU通过地址总线访问任意存储位置
- 存储器内容可被改写→程序可更换

### 1.2 存储器在体系结构中的位置

```
CPU ←→ 存储器（主存RAM） ←→ 外存（硬盘/SSD）
 ↑         ↑                    ↑
寄存器   Cache              非易失存储
(纳秒级)  (SRAM,纳秒)        (毫秒级→微秒级)
```

**冯·诺依曼瓶颈**：CPU和存储器之间只有一条总线，指令和数据共享带宽→速度受限。

## 二、存储单元的基础——从锁存器到寄存器

### 2.1 RS锁存器（RS Latch）——记忆的起点

**原理**：第一个能"记住"状态的电路，输出不仅取决于当前输入，还取决于电路的历史状态。

| R | S | Q⁺ | 功能 |
|---|---|-----|------|
| 0 | 0 | Q | **记忆**（保持） |
| 0 | 1 | 1 | **置位**（Set） |
| 1 | 0 | 0 | **复位**（Reset） |
| 1 | 1 | ? | **禁止**（非法） |

特征方程：**Q⁺ = S + R̄·Q**

> ⚠️ **重难点**：RS=11是非法态，因为两个输出都为0（NOR实现）或都为1（NAND实现），破坏了Q和Q̄互补的约定。

### 2.2 D锁存器（D Latch）——电平触发

- **原理**：用CLK信号控制"何时写入"。CLK=1时透明（输出=D），CLK=0时锁存。
- **问题**：电平触发→CLK=1期间输入变化会直接传导到输出（"透明"问题）。

### 2.3 D触发器（D Flip-Flop）——边沿触发 ★核心

- **原理**：只在时钟**上升沿**（或下降沿）采样并更新输出。
- 解决了D锁存器的透明问题→真正实现"同步时序"。
- **一个D触发器 = 1位存储**。

### 2.4 寄存器（Register）

- **原理**：n个D触发器共享同一个CLK信号→**同时写入n位**。
- 4位寄存器 = 4个D FF + 共用CLK。
- 从课件门库分类：Register属于**MSI（中规模集成电路）**。

> ⚠️ **重难点区分**：
> - **Latch**：电平触发，CLK=1时透明
> - **Flip-Flop**：边沿触发，只在边沿瞬间采样
> - **Register**：多个Flip-Flop的集合

## 三、存储器电路的结构（★★★核心）

### 3.1 一个完整的存储器电路包含

以课件中的4字×3位存储器为例：

```
              ┌─────────────────────────────┐
  地址线 ────→│  Decoder (2:4)              │
  A0, A1      │  选中对应"字"              │
              └──────────┬──────────────────┘
                         ↓ (字选线 Word Line)
              ┌─────────────────────────────┐
  数据输入 ──→│  存储阵列 (4×3 触发器阵列)  │
  I0,I1,I2    │  每个交叉点 = 1位存储       │
              └──────────┬──────────────────┘
                         ↓
              ┌─────────────────────────────┐
              │  三态缓冲器 (Tri-state)     │
              │  控制信号: CS, RD, OE      │
              └──────────┬──────────────────┘
                         ↓
                    数据输出 O0,O1,O2
```

### 3.2 关键控制信号

| 信号 | 全称 | 功能 |
|------|------|------|
| **CS** | Chip Select（片选） | 选中/激活整个芯片 |
| **RD** | Read（读） | 读操作使能 |
| **OE** | Output Enable（输出使能） | 允许数据输出到总线 |
| **WR** | Write（写） | 写操作使能 |

### 3.3 三态缓冲器（Tri-state Buffer）——核心难点 ★

**为什么需要三态？**

> 来自课件：*"输出OR门不能直接连到输出端，否则即使写操作时也会有值输出。"*

三态缓冲器的三种状态：

| D（数据） | C（控制） | S（输出） |
|-----------|-----------|-----------|
| 0 | 0 | **Hi-Z（高阻态）** |
| 0 | 1 | 0 |
| 1 | 0 | **Hi-Z（高阻态）** |
| 1 | 1 | 1 |

- **Hi-Z（高阻态）**：输出端"断开"，相当于这个设备不存在
- **原理意义**：允许多个设备的输出共享同一条数据总线，任何时刻只有一个设备驱动总线

> ⚠️ **重难点**：三态 ≠ 逻辑0！三态意味着"不驱动总线"，在总线上表现为"悬空"。

## 四、存储器扩展（★★★必考）

### 4.1 三种扩展方式

```
原始芯片: 4字 × 3位 (容量=12 bits)

┌──────────────────────────────────────────────────┐
│  位扩展: 增加数据线 → 4字 × 8位                   │
│  方法: 增加5列触发器 + 5对输入/输出线             │
│  地址线不变，数据线从3→8                         │
├──────────────────────────────────────────────────┤
│  字扩展: 增加地址线 → 8字 × 3位                   │
│  方法: 增加4行(4字)×3位 + 1根地址线              │
│  地址线从2→3，用Decoder做片选                    │
├──────────────────────────────────────────────────┤
│  字+位扩展: 同时扩展 → 8字 × 8位                  │
│  方法: 结合以上两种                               │
└──────────────────────────────────────────────────┘
```

### 4.2 字扩展的详细原理（★★★核心）

**场景**：用2片 4字×3位 芯片组成 8字×3位 存储器

```
地址 A2 A1 A0
 │   │  └──┬──┘
 │   │     └→ 片内地址（A1,A0）→ 2:4 Decoder选字
 │   │
 │   └→ A2=0: 选Chip0（0-3号字） → CS0=1
 │      A2=1: 选Chip1（4-7号字） → CS1=1
 │
 └→ 高位地址通过外部Decoder产生片选信号
```

> ⚠️ **关键原理**：扩展存储器时，用Decoder对高位地址译码产生**片选(CS)**信号，低地址用于片内寻址。

### 4.3 位扩展的详细原理

**场景**：用2片 4字×4位 芯片组成 4字×8位 存储器

```
两片芯片共享地址线和控制信号
Chip0 负责 数据线 D[3:0]
Chip1 负责 数据线 D[7:4]
同时读写，数据线宽度翻倍
```

## 五、RAM（随机访问存储器）——活存储器

### 5.1 RAM的定义

> 来自课件：*"Living Memories — commonly known as RAM for Random Access Memory. Access time is the same regardless of which word is requested."*

**Random Access的核心含义**：访问任意地址的时间相同（不像磁带需要顺序寻址）。

**特点**：易失性（volatile）——断电后数据丢失。

### 5.2 SRAM（静态RAM）vs DRAM（动态RAM）（★★★重点对比）

| 特征 | **SRAM** | **DRAM** |
|------|----------|----------|
| **存储单元** | **6个晶体管**（交叉耦合反相器→触发器） | **1个晶体管 + 1个电容** |
| **是否需要刷新** | ❌ **不需要**（触发器自保持） | ✅ **必须刷新**（电容漏电，典型每64ms刷新一次） |
| **速度** | **快**（纳秒级） | 慢（电容充放电慢） |
| **功耗（静态）** | 低 | 需要刷新功耗 |
| **集成度** | 低（6T/bit） | **高**（1T1C/bit） |
| **成本/bit** | 贵 | **便宜** |
| **典型用途** | **Cache（高速缓存）** | **主存（内存条）** |

### 5.3 SRAM存储单元原理（★★）

```
SRAM单元（6T结构）:
      VDD
       │
    ┌──┴──┐
    │ T3  │ T4
    ├──┬──┤
    │  │  │
    │T1│T2│
    │  │  │
    ├──┴──┤
    │ T5  │ T6 ← 字线 (Word Line)
    └──┬──┘
       │
      GND
      
  T1/T2: 交叉耦合反相器（锁存数据）
  T3/T4: 上拉负载
  T5/T6: 访问晶体管（字线控制）
  
  原理: 交叉耦合反相器形成双稳态电路
       一旦写入，只要不断电就永久保持
```

### 5.4 DRAM存储单元原理（★★）

```
DRAM单元（1T1C结构）:
      字线 (Word Line)
          │
    ┌─────┴─────┐
    │           │
  位线 ──── T(晶体管) ──── C(电容)
    
  原理: 
  - 写: 晶体管导通→电容充电/放电(1/0)
  - 读: 晶体管导通→电容电荷流向位线→检测电压
  - 问题: 读操作是破坏性的(电荷流失),需回写
  - 刷新: 电容漏电→需定期重写
```

> ⚠️ **DRAM的核心矛盾**：
> - 集成度高（单元小→容量大→便宜） ← 优势
> - 需要刷新+读写慢+破坏性读出 ← 代价
>
> 这就是为什么**Cache用SRAM（快但贵），主存用DRAM（慢但便宜）**。

### 5.5 DRAM的地址组织（RAS/CAS）

> 来自课件：*"Row/Column selection: Raw Address Strobe, Column Address Strobe"*

大容量DRAM将地址分成**行地址**和**列地址**，分时复用地址线：

```
地址线 A[10:0] ──→ 先送行地址(RAS选通) → 再送列地址(CAS选通)
                          ↓                       ↓
                    选中一整行(如2048位)    从中选出具体位
```

- 好处：减少引脚数（11根地址线可寻址2048×2048=4M位）
- 例如课件中：2048×2048 的 1位×4M 存储器，行地址和列地址各11位

## 六、ROM（只读存储器）——死存储器

### 6.1 ROM的定义

> 来自课件：*"Dead Memories — Read-only memory is not volatile, unlike RAM, i.e. it does not lose its data in the absence of power. Used to store the BIOS."*

**特点**：非易失性（non-volatile）——断电不丢失数据。

### 6.2 ROM的分类（按可编程性）

| 类型 | 全称 | 编程方式 | 擦除方式 | 可重写次数 |
|------|------|----------|----------|-----------|
| **ROM** | Read-Only Memory | 工厂掩膜 | 不可擦 | 0 |
| **PROM** | Programmable ROM | 用户烧熔丝 | 不可擦 | 1次(OTP) |
| **EPROM** | Erasable PROM | 用户编程 | **紫外线**擦除 | 多次(需拆下芯片) |
| **EEPROM** | Electrically EPROM | 用户编程 | **电擦除** | 多次(在线擦写) |
| **Flash** | Flash Memory | 用户编程 | 电擦除(块擦除) | 多次(有限寿命) |

### 6.3 ROM实现组合逻辑的原理（★★思维拓展）

> 来自课件：*"A combinatorial logic system can be realized with a ROM since the state of the outputs depends only on the state of the inputs. Input = address, Output = data."*

```
ROM实现组合逻辑:
  输入变量 → 地址线（A0, A1, ...）
  输出函数 → 数据线（D0, D1, ...）
  
  真值表的每一行 = ROM中的一个存储单元
  例如: 3输入多数函数
  Address | Data
  000     | 0
  001     | 0
  010     | 0
  011     | 1  ← 存储真值表的输出
  ...
```

**ROM vs PLA实现组合逻辑**：
- ROM：用全译码器→2ⁿ个地址全生成→简单但浪费资源
- PLA：只生成需要的乘积项→灵活但两层可编程延迟大

## 七、SSD（固态硬盘）

### 7.1 SSD技术基础

> 来自课件：*"Today, SSD hard drives are becoming more and more popular and are overshadowing classic mechanical hard drive technology."*

### 7.2 SSD vs HDD 对比

| 特征 | **SSD（固态硬盘）** | **HDD（机械硬盘）** |
|------|---------------------|---------------------|
| **技术** | NAND Flash（EEPROM的进化） | 磁性盘片 + 机械臂 |
| **访问时间** | **~0.1ms** | 数ms（寻道+旋转延迟） |
| **读写速度** | **数GB/s** | 取决于转速(5400/7200rpm) |
| **功耗** | 低 | 高（电机驱动） |
| **噪音** | 无（无机械部件） | 有 |
| **可靠性** | 高（无机械故障） | 较低（机械磨损） |
| **写入寿命** | **有限擦写次数** | 理论上无限 |
| **容量成本** | 仍较贵 | 便宜（大容量） |

### 7.3 NAND Flash的工作原理（★★）

```
NAND Flash存储单元（浮栅晶体管）:
          控制栅 (Control Gate)
              │
         ═════╪═════  (氧化层)
              │
        ┌─────┴─────┐
        │  浮栅(FG)  │  ← 电子困在浮栅中→数据保持
        └─────┬─────┘
              │
         ═════╪═════  (隧道氧化层)
              │
          源极───────漏极
          
  写: 隧道效应→电子注入浮栅→阈值电压改变→存0
  擦: 反向电压→电子从浮栅中移除→恢复→存1
  读: 测量阈值电压→判断0或1
```

> ⚠️ **SSD的核心矛盾**：
> - 写入需要高电压（隧道效应）→氧化层逐渐劣化 → **擦写次数有限**（典型：数百~数万次）
> - 擦除以**块**（Block）为单位，写入以**页**（Page）为单位 → 需要**垃圾回收**（GC）和**磨损均衡**（Wear Leveling）
> - 这些由SSD控制器（FTL, Flash Translation Layer）管理

## 八、存储器层次结构总览（★★★综合理解）

```
        速度         容量       成本/bit      技术
        ────        ────       ────────      ────
CPU ←→ 寄存器       ~1KB        最贵           
        ↑  <1ns
CPU ←→ Cache(L1)   ~64KB       很贵          SRAM (6T)
        ↑ ~1ns
CPU ←→ Cache(L2/L3) ~MB        贵            SRAM (6T)
        ↑ ~10ns
CPU ←→ 主存(内存)   ~GB         中等          DRAM (1T1C)
        ↑ ~50-100ns
CPU ←→ 固态硬盘(SSD) ~TB        便宜          NAND Flash
        ↑ ~0.1ms
CPU ←→ 机械硬盘(HDD) ~TB        最便宜        磁性介质
        ↑ ~数ms
```

**局部性原理**决定了这种层次结构的有效性：
- **时间局部性**：刚访问过的数据很可能再次访问 → Cache
- **空间局部性**：刚访问过的数据附近的数据很可能被访问 → Cache Line/Cache Block

## 九、重难点总结速查

| 难点 | 核心要点 |
|------|----------|
| **SRAM vs DRAM单元** | SRAM=6T触发器（自保持），DRAM=1T1C（需刷新） |
| **刷新机制** | DRAM电容漏电→必须定期（~64ms）重写所有行 |
| **三态缓冲器** | Hi-Z≠逻辑0，允许多设备共享总线 |
| **存储器扩展** | 字扩展=增加地址线+Decoder片选；位扩展=增加数据线 |
| **RAS/CAS** | DRAM地址复用：先送行地址再送列地址，节省引脚 |
| **ROM实现逻辑** | 输入→地址，输出→数据，真值表直接映射 |
| **SSD写入限制** | 隧道效应导致氧化层劣化→有限P/E cycle |
| **Cache原理** | SRAM实现，利用局部性原理，CPU和主存间的速度桥梁 |

---

### 一句话记忆

> **SRAM快但贵→Cache；DRAM慢但便宜→主存；ROM断电不丢→BIOS/固件；SSD=NAND Flash→快但有限寿命；扩展=Decoder片选+增加地址/数据线。**
